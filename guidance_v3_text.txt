4.  The Workflow â€” How Every Module Is Built
This is the complete operating procedure. Follow it for every module from 02 to 25 without exception.

Step 1 â€” Open Antigravity and go to the evermaze folder
All work happens inside the evermaze folder. Never create a new folder for a new module. Modules add files to the same folder or replace existing files within it.

Step 2 â€” Open Antigravity's built-in AI assistant
Antigravity has an AI coding assistant built in. This replaces Google AI Studio entirely. You use this AI to generate code for each module.

Step 3 â€” Start with the Master Prompt
Every AI session â€” without exception â€” begins by pasting the full contents of MASTER_PROMPT.md into the Antigravity AI assistant. This gives it the full project context: architecture, naming rules, constants, event names, code standards.
Never skip the Master Prompt. An AI session without it does not know the project rules, will name things wrong, and will create code that breaks other modules.

Step 4 â€” Paste the module system prompt
Immediately after the Master Prompt, paste the system prompt for the current module. The system prompt tells the AI exactly what to build: which files to create, which files to modify, the functional requirements, the edge cases, and the testing checklist.

Step 5 â€” Wait for confirmation before code
After pasting both prompts, the AI must respond with a confirmation listing:
â€¢  The module name and number it is building
â€¢  The exact files it will create or modify
â€¢  Any questions or ambiguities before it starts

If the AI skips this and starts writing code immediately â€” type "Stop. List the files you are about to create before writing any code." Do not accept code without a scope confirmation first. This prevents the AI from building outside its assigned module.

Step 6 â€” Review and approve
Compare the AI's listed files against the SCOPE section of the system prompt. If they match â€” type "Confirmed. Proceed."
If they do not match â€” correct it before code is written. Example: "File X is not in the scope. Remove it from your plan and confirm again."

Step 7 â€” Receive the code
The AI writes each file. Because you are inside Antigravity, you apply each file directly:
â€¢  If the file does not exist yet: create it at the correct path and paste the content
â€¢  If the file already exists (a stub from Module 01): open it and replace all content with the new version
â€¢  Do not modify the code before applying it unless there is an obvious syntax error

Step 8 â€” Run npm install if needed
If the module added new packages to package.json, run these in the Antigravity terminals:
Terminal 1:  cd server && npm install && npm run dev
Terminal 2:  cd client && npm install && npm run dev
If no new packages were added, just restart both servers.

Step 9 â€” Run the testing checklist
Every system prompt ends with a testing checklist of 8â€“12 items. Run every single one manually inside Antigravity.
â€¢  Open the preview URL in a browser tab
â€¢  Open a second tab for multiplayer tests
â€¢  Each item: PASS or FAIL â€” no partial credit
All items must PASS before the next module begins. If one item fails, the module is not complete. Fix it first.

Step 10 â€” Fix failures in the same session
If a test fails, stay in the same Antigravity AI session and type:
"Testing checklist item [number] is failing. Exact error: [paste error here]. Fix only this issue. Do not change files outside the current module scope."
Apply the fix. Re-run only the failing test. Repeat until it passes.

Step 11 â€” Report back to Claude
Come back to this conversation and report using this exact format:
Module [number] â€” [name]: COMPLETE
All [N] checklist items: PASS
Errors encountered and fixed: [describe or "none"]
Files created or modified: [list them]
Ready for: Module [next number]

Step 12 â€” Receive the next system prompt from Claude
Claude writes each system prompt only after the previous module is verified. This ensures each prompt is written with full knowledge of what was actually built. You will receive the next system prompt here, copy it, and use it in Antigravity for the next session.

System prompts for modules 03â€“25 are written one at a time by Claude. They are not pre-written. This is intentional â€” each prompt depends on what was actually built before it.
